# portal-pc-resource

## 项目简介

`portal-pc-resource` 是一个用于归档与发布独立 HTML 页面的仓库。本项目不针对任何特定域名或网站，仅作为静态网页资源的统一存放与版本管理空间。所有页面均为独立构建，适合快速部署、分发或作为参考素材使用。

## 目录说明

```
portal-pc-resource/
├── pages/          # 存放所有独立 HTML 页面，按用途或时间组织子目录
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 仓库首页/导航页（可选）
└── README.md       # 本文件
```

- `pages/`：每个子目录对应一个独立页面，包含完整的 HTML 文件及其依赖资源。
- `assets/`：存放多个页面共用的样式、脚本或媒体文件，便于统一维护。
- `index.html`：可选的仓库入口页，用于展示页面列表或快速跳转。

## 页面归档说明

归档的 HTML 页面具有以下特点：

- **独立性**：每个页面自成一体，不依赖外部服务或特定域名。
- **静态性**：页面内容以纯静态 HTML/CSS/JS 构成，无需后端支持。
- **版本化**：通过 Git 提交记录跟踪页面变更，方便回溯历史版本。
- **可部署**：可直接将 `pages/` 目录部署至任意静态托管服务（如 GitHub Pages、Netlify 等）。

## 维护说明

- 新增页面时，请在 `pages/` 下创建独立子目录，并确保所有资源文件路径相对。
- 若页面使用公共资源，请优先引用 `assets/` 中的文件，避免重复。
- 修改已有页面时，请保持向后兼容或明确标注版本变更。
- 不建议在页面内嵌入跨域远程脚本或外部资源，以维持归档的长期可用性。
